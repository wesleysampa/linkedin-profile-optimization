# 🧠 PROFESSIONAL_CONTEXT — Guia Estratégico

O bloco `<PROFESSIONAL_CONTEXT>` é o ponto mais importante de todo o processo.

Ele define:

* Como seu perfil será interpretado
* Para quais vagas você vai aparecer
* Como recrutadores e IA vão te enxergar

> ⚠️ Se este bloco estiver errado, todo o restante perde qualidade.

---

## 🎯 Antes de preencher (leia isso)

Não responda pensando no seu currículo.

Responda pensando:

👉 **Como você quer ser encontrado no mercado**

---

## 🧩 Estrutura do PROFESSIONAL_CONTEXT

---

## 🔹 MAIN_DOMAIN

Define sua **área principal em TI**.

Pergunta:

> Se só pudesse escolher uma área, qual seria?

### Exemplos

* Infraestrutura
* Cloud & Platform Engineering
* Segurança da Informação
* Dados & Analytics
* Desenvolvimento de Software
* Governança / Gestão de TI

---

## 🔹 SUBDOMAIN

Define suas **especialidades técnicas**.

### Como preencher

* Do mais estratégico → mais técnico
* Até 3 ou 4 blocos
* Separados por `/`

### Exemplos

* Cloud (Azure) / Windows Server / Virtualização
* DevOps / CI-CD / Observabilidade
* Security / Cloud Security / IAM

---

## 🔹 BALANCE

Define seu **posicionamento entre Cloud e On-Prem**.

Isso evita que você seja visto como:

* muito legado
* ou sem experiência prática

### Opções

* Cloud-first
* Cloud + On-Prem (equilíbrio) ✅
* On-Prem com foco em modernização
* Cloud native

---

## 🔹 CAREER_TRACK

Define os **tipos de problema que você resolve**.

Isso vai guiar:

* seu About
* suas experiências
* sua narrativa

### Exemplos

* Operações
* Arquitetura
* Modernização
* Migração
* Governança
* Segurança
* Observabilidade
* Automação

---

## 🔹 SENIORITY_SCOPE

Define como o mercado deve te avaliar.

### Regras importantes

* Não inclua Júnior se não for
* Use `|` para progressão

### Exemplos

* Pleno | Sênior
* Sênior | Especialista ✅
* Especialista | Principal

---

## 🔹 TARGET_MARKETS

Define onde você quer atuar.

Inclua:

* localização
* modelo de trabalho

### Exemplos

* Brasil (SP), Remoto
* Brasil | Internacional (Remoto)
* LATAM, Híbrido

---

## 🔹 TARGET_ENVIRONMENTS

Define em que tipo de empresa você gera mais valor.

### Exemplos

* Startups
* Enterprise ✅
* Ambientes regulados (Saúde / Financeiro) ✅
* Missão crítica
* Alta disponibilidade

---

## 🔹 LANGUAGE_PREF

Idioma dos textos finais.

### Opções

* pt-BR
* en-US
* bilingual (pt/en)

> 💡 Termos técnicos em inglês são recomendados mesmo em pt-BR

---

# ✅ EXEMPLO COMPLETO (IMPORTANTE)

```xml
<PROFESSIONAL_CONTEXT>

  <MAIN_DOMAIN>Cloud & Platform Engineering</MAIN_DOMAIN>

  <SUBDOMAIN>
    Cloud (Azure) / Infraestrutura / Automação
  </SUBDOMAIN>

  <BALANCE>Cloud + On-Prem (equilíbrio)</BALANCE>

  <CAREER_TRACK>
    <TRACK>Operações</TRACK>
    <TRACK>Modernização</TRACK>
    <TRACK>Automação</TRACK>
  </CAREER_TRACK>

  <SENIORITY_SCOPE>Sênior | Especialista</SENIORITY_SCOPE>

  <TARGET_MARKETS>Brasil | Remoto</TARGET_MARKETS>

  <TARGET_ENVIRONMENTS>
    <TYPE>Enterprise</TYPE>
    <TYPE>Ambientes regulados</TYPE>
  </TARGET_ENVIRONMENTS>

  <LANGUAGE_PREF>pt-BR</LANGUAGE_PREF>

</PROFESSIONAL_CONTEXT>
```

---

# 🔍 Como saber se está bem definido?

Faça esse checklist:

* Seu foco está claro em 1 área?
* Suas especialidades fazem sentido juntas?
* Seu nível (Pleno/Sênior/etc.) está coerente?
* Você consegue explicar isso em 1 frase?

Se a resposta for “não” em algum ponto:

👉 ajuste antes de continuar

---

# ⚠️ Erros comuns

* Misturar muitas áreas (ex: Dev + Infra + Dados)
* Usar termos genéricos (“TI”, “Tecnologia”)
* Não definir senioridade
* Não pensar no mercado-alvo
* Preencher com base no passado, não no objetivo

---

# 🎯 Conclusão

O PROFESSIONAL_CONTEXT não é um formulário.

👉 É um posicionamento estratégico.

Se bem definido, todo o restante do processo funciona melhor:

* análise
* otimização
* visibilidade
