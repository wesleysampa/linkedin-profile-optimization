# PROFESSIONAL_CONTEXT — Guia Estratégico

O bloco `<PROFESSIONAL_CONTEXT>` é o **briefing estratégico** do perfil.
Ele define como a IA deve **“enxergar” o profissional**, para quem comunicar
e com qual linguagem.

Se este bloco estiver mal definido, todo o processo será comprometido.

---

## MAIN_DOMAIN

Define a **grande área de atuação em TI**.

Pergunta-chave:
> Se um recrutador tivesse que colocar seu perfil em **uma única caixa**, qual seria?

### Exemplos válidos
- Infraestrutura  
- Cloud & Platform Engineering  
- Segurança da Informação  
- Dados & Analytics  
- Desenvolvimento de Software  
- Governança / Gestão de TI  

---

## SUBDOMAIN

Define **especialidades técnicas e foco de atuação**.

### Como preencher
- Sempre do **mais estratégico → mais técnico**
- Até **3 ou 4 blocos**
- Separados por `/`

### Bons exemplos
- Cloud (Azure) / Windows Server / Virtualização  
- DevOps / CI-CD / Observabilidade  
- Security / Cloud Security / IAM  

---

## BALANCE

Define o **posicionamento Cloud x On-Prem**.

Este campo protege sua **senioridade**.

### Opções comuns
- Cloud-first  
- Cloud + On-Prem (equilíbrio) ✅  
- On-Prem com foco em modernização  
- Cloud native  

---

## CAREER_TRACK

Define **quais tipos de problemas você resolve** no dia a dia.
Guia o storytelling, os bullets da experiência e o About.

### Exemplos de TRACK
- Operações  
- Sustentação  
- Arquitetura  
- Modernização  
- Migração  
- Governança  
- Segurança  
- Observabilidade  
- Automação  

---

## SENIORITY_SCOPE

Define **como você deve ser avaliado pelo mercado**.

### Regras
- Nunca inclua Júnior, se não for
- Use `\` para progressão natural

### Exemplos corretos
- Pleno \ Sênior  
- Sênior \ Especialista ✅  
- Especialista \ Principal  

---

## TARGET_MARKETS

Define **onde você quer ser encontrado**.

Inclua:
- País / cidade (quando relevante)
- Modelo: Remoto / Híbrido / Presencial

### Exemplos
- Brasil (SP), Remoto  
- Brasil \ Internacional (Remoto)  
- LATAM, Híbrido  

---

## TARGET_ENVIRONMENTS

Define **em que tipo de empresa você performa melhor**.

### Exemplos
- Startups  
- Enterprise ✅  
- Ambientes regulados (Saúde / Financeiro) ✅  
- Missão crítica  
- Alta disponibilidade  

Este campo **aumenta muito a percepção de valor** para headhunters.

---

## LANGUAGE_PREF

Define o idioma dos textos finais.

### Opções
- pt-BR  
- en-US  
- bilingual (pt/en)  

> Mesmo com pt-BR, termos técnicos em inglês são desejáveis.
